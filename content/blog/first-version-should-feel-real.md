我后来意识到，所谓“先把功能做出来，样式以后再说”，对长期个人项目往往是一种拖延的变体。它看起来务实，实际上是在把最影响持续投入意愿的部分往后推。

对我来说，这个博客不是临时练手页，而是会持续记录项目、笔记和思考的地方。既然它本身也是作品的一部分，那第一版就不能只像一个占位页面。

> 第一版不一定要极致，但一定要已经具备“我愿意公开给别人看”的完成感。

这并不意味着过度打磨每个细节，而是先把大方向做对：排版舒服、层次清楚、风格统一、页面之间像同一个网站长出来的。

```rust
fn main() {
    println!("先把结构和方向做对，再继续打磨细节。");
}
```

我希望这个站点以后每次回头看，看到的不是“当时凑合的壳子”，而是一套从第一版就已经值得被继续完善的基础。
