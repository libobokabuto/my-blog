以前我会把“项目代码”“学习笔记”“产品文档”拆得到处都是。短期看上去很清爽，长期却很容易失去上下文。

这次我决定把它们都放在一个仓库里：

- `PRDS/` 放需求和开发计划
- `LEARNING/` 放围绕真实任务的学习记录
- `content/` 放博客和后续内容
- 正式代码和部署配置放在同一个根目录下

这样做的好处不是“目录更整齐”，而是每一次实现都能对回原始意图，每一次学习也都能落回真实任务。

当这些东西挨在一起时，项目终于不再像一次性练手，而更像一个会持续生长的网站产品。
