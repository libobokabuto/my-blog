# 第四版开发计划
## 1. 当前状态
状态：进行中

当前第四版已经完成从“纯规划”到“正式骨架实现”的切换：
- 阶段一“数据层与内容后台”已完成第一批正式骨架
- 阶段二“搜索与索引基础设施”已完成第一批正式骨架
- 阶段三“统计与任务系统”已进入进行中
- 阶段四“外部同步与开放能力”已进入进行中

## 2. 开发目标
在第三版已完成的静态内容系统基础上，推进最小后端能力闭环，包括：

1. 数据层
2. 后台入口
3. 搜索索引基础设施
4. 统计持久化基础
5. 任务与同步基础

## 3. 开发策略
第四版建议采用以下策略推进：

1. 先定服务边界和数据边界，再写页面
2. 先做后台读能力和治理能力，再逐步引入写能力
3. 先做最小内部使用闭环，再考虑外部开放
4. 先保证结构可演进，再追求完整功能
5. 当前数据库统一采用 MySQL，Redis 只承担运行态和轻量状态记录

## 4. 阶段划分
### 阶段一：数据层与内容后台骨架

目标：
- 明确内容数据模型
- 建立服务端聚合与持久化边界
- 建立后台入口和内容治理页面骨架

状态：进行中

当前完成情况：
1. 已定义统一内容实体、标签、系列、关联、状态字段骨架
2. 已定义内容读模型与后台查询模型
3. 已落地 `/admin`、`/admin/content`、`/admin/content/:id`
4. 已预留后续写入与发布流转接口边界
5. 已明确第三版内容系统到第四版后台读模型的衔接层
6. 已把 `notes.board` 作为正式内容字段保留给后续索引和治理复用

### 阶段二：搜索与索引基础设施

目标：
- 建立索引持久化或索引元信息存储结构
- 建立搜索重建流程
- 建立服务端搜索接口骨架

状态：进行中

当前完成情况：
1. 已设计搜索文档模型
2. 已设计索引构建与重建流程骨架
3. 已落地 `/admin/search`
4. 已兼容第三版现有 `/search` 前台入口
5. 已明确 MySQL 与 Redis 在搜索基础设施中的职责边界
6. 已把 `notes.board` 纳入索引字段与诊断视图
7. 已在 MySQL 中初始化 `search_documents` 与 `search_rebuild_runs` 表

### 阶段三：统计与任务系统

目标：
- 建立持久化统计结构
- 建立任务定义、任务执行与任务记录边界
- 接入统计面板与任务面板

状态：进行中

当前完成情况：
1. 已定义统计快照模型与任务运行记录模型
2. 已落地 `/admin/stats` 与 `/admin/tasks`
3. 已建立 `stats_snapshots` 与 `task_runs` MySQL 表
4. 已让搜索重建与同步占位运行共用统一任务记录边界

### 阶段四：外部同步与开放能力

目标：
- 建立外部同步边界
- 建立最小开放接口结构
- 为第五版预留认证与更复杂集成能力

状态：进行中

当前完成情况：
1. 已定义同步源模型与同步运行记录模型
2. 已落地 `/admin/sync`
3. 已建立 `sync_sources` 与 `sync_runs` MySQL 表
4. 已预置 `notes-import`、`project-catalog`、`search-runtime` 三个同步边界
5. 当前仍是占位同步运行，不接入真实第三方 API

## 5. 建议优先顺序
建议第四版按以下顺序继续推进：

1. 先完成搜索索引端到端重建验证
2. 再把统计快照和任务记录串到真实后台动作
3. 最后把同步边界从占位源推进到真实导入器

当前建议：
- 先解决本机 `cargo run -p server --bin server` 的 Rust 目标环境问题，完成 `/admin/search` 与 `/admin/sync` 的真实运行验证
- 然后补首轮索引重建记录、任务记录和同步运行记录
- 再评估是否需要引入队列，把重建与同步从应用内同步执行升级为异步任务

## 6. 前置条件与基础设施
第四版继续推进时需要先确认以下基础设施方案，但不要直接替用户创建：

1. MySQL
2. Redis
3. 是否需要对象存储
4. 是否需要队列
5. 本地开发与部署环境的配置方式

当前已确认：
- MySQL：已创建 `my-blog` 库
- Redis：已可本机运行
- 对象存储：当前阶段不必立即引入
- 队列：当前阶段可暂缓，等搜索重建和同步真正变异步时再引入

## 7. 完成判断
第四版可以认为完成，当以下条件同时满足：

1. 已建立正式内容数据层
2. 已建立最小后台治理入口
3. 搜索、统计、任务、同步都有服务端边界
4. MySQL 与 Redis 的运行态链路至少完成一轮真实验证
5. 第五版可以不推翻第四版结构继续演进
