# 第四版开发计划
## 1. 当前状态
状态：未开始

当前第四版尚未进入正式实现，已先完成总 PRDS 骨架设计。

## 2. 开发目标
在第三版已完成的静态内容系统基础上，推进最小后端能力闭环，包括：

1. 数据层
2. 后台入口
3. 搜索索引基础设施
4. 统计持久化基础
5. 任务与同步基础

## 3. 开发策略
第四版建议采用以下策略推进：

1. 先定服务边界和数据边界，再写页面
2. 先做后台读能力和治理能力，再逐步引入写能力
3. 先做最小内部使用闭环，再考虑外部开放
4. 先保证结构可演进，再追求完整功能

## 4. 阶段划分
### 阶段一：数据层与内容后台骨架

目标：
- 明确内容数据模型
- 建立服务端聚合与持久化边界
- 建立后台入口和内容治理页面骨架

状态：未开始

当前子任务：
1. 定义内容实体、标签、系列、关联、状态字段
2. 定义内容读模型与后台查询模型
3. 设计 `/admin` 与 `/admin/content`
4. 预留后续写入与发布流转接口

### 阶段二：搜索与索引基础设施

目标：
- 建立索引持久化或索引元信息存储结构
- 建立搜索重建流程
- 建立服务端搜索接口骨架

状态：未开始

当前子任务：
1. 设计搜索文档模型
2. 设计索引构建与重建流程
3. 设计后台搜索诊断页
4. 兼容第三版现有搜索入口

### 阶段三：统计与任务系统

目标：
- 建立持久化统计结构
- 建立任务定义、任务执行与任务记录边界
- 接入统计面板与任务面板

状态：未开始

当前子任务：
1. 设计访问统计和内容活跃统计结构
2. 设计任务表述模型
3. 设计 `/admin/stats` 与 `/admin/tasks`
4. 区分同步任务、索引任务、统计任务

### 阶段四：外部同步与开放能力

目标：
- 建立外部同步边界
- 建立最小开放接口结构
- 为第五版预留认证与更复杂集成能力

状态：未开始

当前子任务：
1. 设计同步源模型
2. 设计同步任务流转
3. 设计 `/admin/sync`
4. 设计内部 API 与未来开放 API 的边界

## 5. 建议优先顺序
建议第四版按以下顺序推进：

1. 数据层与后台骨架
2. 搜索索引基础设施
3. 统计与任务系统
4. 外部同步与开放能力

## 6. 前置条件与基础设施
第四版实现前需要先确认以下基础设施方案，但不要直接替用户创建：

1. 数据库选型
2. 缓存选型
3. 是否需要对象存储
4. 是否需要队列
5. 本地开发与部署环境的配置方式

## 7. 完成判断
第四版可以认为完成，当以下条件同时满足：

1. 已建立正式内容数据层
2. 已建立最小后台治理入口
3. 搜索、统计、任务、同步都有服务端边界
4. 第五版可以不推翻第四版结构继续演进
