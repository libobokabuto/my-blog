# 第四版 - 统计与任务系统 PRD
## 1. 功能定位
这个模块负责让站点从第三版的轻量静态统计，升级到最小持久化统计与后台任务系统。

## 2. 当前版本状态
状态：进行中

当前判断：
- 已正式开始第四版第三项
- 当前阶段先做治理指标和后台任务记录，不直接做公开流量平台

## 3. 当前实现范围
1. 统计快照模型
2. 任务模型与任务记录
3. 统计聚合接口
4. 任务执行与任务状态接口
5. `/admin/stats`
6. `/admin/tasks`

## 4. 后端边界说明
这个模块已经明确属于：
1. 持久化统计
2. 任务记录
3. 后台运行状态管理

当前阶段的实际边界说明：
- 统计先落地为 MySQL 快照，不做复杂时间序列
- 任务先落地为应用内触发记录，不引入独立 worker
- 搜索重建与同步运行先共用统一任务边界

## 5. 暂不要求的内容
1. 复杂 BI 平台
2. 实时流处理
3. 分布式任务调度平台

## 6. 当前实现结果
1. 已建立 `MetricSnapshot` 与 `TaskRunRecord`
2. 已落地 `/admin/stats`
3. 已落地 `/admin/tasks`
4. 已建立统计快照和任务记录服务端接口骨架
5. 已在 MySQL 中初始化 `stats_snapshots` 与 `task_runs`
