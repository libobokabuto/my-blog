# 第四版 - 统计与任务系统开发计划
## 1. 当前状态
状态：进行中

## 2. 开发目标
建立最小可用的持久化统计与任务边界，为搜索重建、同步、治理提供统一执行基础。

## 3. 建议实施顺序
1. 先定义统计结构
2. 再定义任务结构
3. 再接入后台页
4. 最后让搜索和同步复用任务能力
5. 再补真实运行记录

## 4. 具体任务
1. 设计治理统计、更新时间统计、运行次数统计
2. 设计任务类型、任务状态、执行记录
3. 设计 `/admin/stats`
4. 设计 `/admin/tasks`
5. 让搜索重建和同步运行写入统一任务表

## 5. 完成标准
1. 有最小持久化统计
2. 有统一任务边界
3. 搜索和同步可以挂接到任务系统

## 6. 当前完成情况
1. 已完成统计快照模型与任务记录模型
2. 已完成 `/admin/stats` 与 `/admin/tasks`
3. 已完成 `stats_snapshots` 与 `task_runs` MySQL 表初始化
4. 当前剩余关键工作是补第一次真实任务运行记录
