# 第四版 - 搜索与索引基础设施 PRD
## 1. 功能定位
这个模块负责把第三版的静态搜索升级为可重建、可持久化、可诊断的搜索基础设施。

## 2. 当前版本状态
状态：进行中

当前判断：
- 已正式开始第四版第二项
- 当前阶段先把第三版现有搜索能力桥接到“可持久化索引 + 可诊断后台视图”
- MySQL 负责持久化搜索文档与重建记录
- Redis 负责记录重建状态与轻量运行态信息

## 3. 当前实现范围
1. 搜索文档模型
2. 搜索索引持久化
3. 搜索构建与重建流程骨架
4. 服务端搜索接口骨架
5. 后台搜索诊断入口
6. 与第三版前台 `/search` 的兼容衔接

当前第一批实现补充：
- 搜索文档至少覆盖标题、摘要、标签、正文纯文本、内容类型、slug
- `notes.board`、`notes.stage` 进入正式索引字段
- `/admin/search` 可以看到索引状态、最近重建记录、示例查询诊断

## 4. 后端边界说明
这个模块已经进入：
1. 索引持久化
2. 服务端查询
3. 后台运维与诊断

当前阶段的实际边界说明：
- MySQL 用于存储搜索文档和重建记录
- Redis 用于重建状态与轻量运行时状态
- 前台搜索会优先保持可用，再逐步切换到底层持久化索引

## 5. 暂不要求的内容
1. 复杂推荐算法
2. 向量检索
3. 多语言搜索平台化
4. 复杂分布式索引集群

## 6. 当前实现结果
1. 已建立搜索文档模型、搜索重建记录模型与查询诊断模型
2. 已建立 `/admin/search`
3. 已建立搜索重建入口骨架
4. 已建立 MySQL 持久化表结构自举逻辑
5. 已建立 Redis 运行态状态写入逻辑
6. 已把 `notes.board` 纳入索引字段
7. 当前前台 `/search` 可在持久化索引不可用时回退到第三版实时搜索
8. 当前 MySQL `my-blog` 库中已初始化 `search_documents` 与 `search_rebuild_runs`
