# 第四版 - 数据层与内容后台开发计划
## 1. 当前状态
状态：未开始

## 2. 开发目标
建立第四版最核心的服务端基础，让内容系统从“构建期聚合”进入“正式数据层 + 后台治理入口”阶段。

## 3. 建议实施顺序
1. 先定义内容数据模型
2. 再定义服务端查询接口
3. 再搭建后台页面骨架
4. 最后预留写入、发布、治理动作接口

## 4. 具体任务
1. 设计内容实体、状态字段、关联结构
2. 设计内容后台聚合视图模型
3. 设计 `/admin`、`/admin/content`、`/admin/content/:id`
4. 设计内容治理动作的接口边界

## 5. 完成标准
1. 有正式内容数据结构
2. 有后台内容入口
3. 有服务端聚合接口
4. 后续搜索、统计、同步可以复用这里的数据边界
