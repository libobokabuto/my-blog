# 第四版 - 数据层与内容后台 UI
## 1. 当前 UI 状态
状态：进行中

## 2. 视觉方向
后台 UI 不追求复杂运营平台风格，建议维持：

1. 清晰信息密度
2. 稳定表格与侧栏结构
3. 以状态、类型、问题项为重点

## 3. 页面气质
应更像“个人内容控制台”，而不是通用 SaaS 后台。

## 4. 页面建议
1. `/admin` 显示系统概览
2. `/admin/content` 显示内容列表、筛选、状态
3. `/admin/content/:id` 显示内容详情、关联、治理项

第一批实现补充：
- `/admin` 更像“系统概览 + 模块入口”
- `/admin/content` 强调跨内容类型统一视图，而不是 blog / notes / projects 三套后台
- `/admin/content/:id` 先展示来源、状态、标签、系列/阶段/项目状态、关联项与问题摘要

## 5. 后续优化方向
1. 增加内容问题提示
2. 增加关联关系可视化
3. 增加发布流转状态展示
4. 在用户准备数据库后，切换为真正可编辑的后台详情页

## 6. 当前落地结果
1. `/admin` 已落地为系统概览页
2. `/admin/content` 已落地为统一内容列表页
3. `/admin/content/:id` 已落地为内容详情与治理摘要页
4. 当前视觉重点已经从公开展示页切到“控制台式信息读取”
