# 第四版 - 数据层与内容后台 UI
## 1. 当前 UI 状态
状态：未开始

## 2. 视觉方向
后台 UI 不追求复杂运营平台风格，建议维持：

1. 清晰信息密度
2. 稳定表格与侧栏结构
3. 以状态、类型、问题项为重点

## 3. 页面气质
应更像“个人内容控制台”，而不是通用 SaaS 后台。

## 4. 页面建议
1. `/admin` 显示系统概览
2. `/admin/content` 显示内容列表、筛选、状态
3. `/admin/content/:id` 显示内容详情、关联、治理项

## 5. 后续优化方向
1. 增加内容问题提示
2. 增加关联关系可视化
3. 增加发布流转状态展示
