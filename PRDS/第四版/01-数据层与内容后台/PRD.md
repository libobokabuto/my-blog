# 第四版 - 数据层与内容后台 PRD
## 1. 功能定位
这个模块负责把第三版的静态内容聚合，推进为第四版的正式内容数据层与后台治理入口。

它解决的问题是：
- 内容越来越多后，单靠构建期聚合不够稳定
- 内容治理缺少统一后台入口
- 后续搜索、统计、同步都需要正式数据边界

## 2. 当前版本状态
状态：未开始

## 3. 当前实现范围
本模块第四版应覆盖：

1. 内容实体模型
2. 标签、系列、归档、关联的正式数据关系
3. 内容状态字段
4. 后台内容列表与详情骨架
5. 服务端内容查询与聚合接口

## 4. 后端边界说明
这个模块已经明确触及第四版后端边界：

1. 数据库 schema
2. 服务端读写逻辑
3. 后台管理入口

## 5. 暂不要求的内容
1. 完整多角色权限
2. 富文本在线编辑器
3. 复杂审核流
4. 面向外部用户的内容协作
