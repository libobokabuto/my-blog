# 第四版 - 外部同步与开放能力 PRD
## 1. 功能定位
这个模块负责为博客建立最小外部同步边界与开放接口边界，让后续版本能够接入更多外部来源和外部消费方式。

## 2. 当前版本状态
状态：进行中

当前判断：
- 已正式开始第四版第四项
- 当前阶段只建立同步边界和运行记录，不直接进入真实开放平台

## 3. 当前实现范围
1. 同步源模型
2. 同步任务边界
3. 外部内容或项目数据导入流程骨架
4. 内部 API 与未来开放 API 的边界
5. `/admin/sync`

## 4. 后端边界说明
这个模块明确进入：
1. 外部系统集成边界
2. 同步运行记录
3. API 边界设计

当前阶段的实际边界说明：
- 同步先落地为源登记、占位运行和任务记录
- 真实第三方接入、认证、开放平台保留到后续阶段

## 5. 暂不要求的内容
1. 大规模第三方平台分发
2. 复杂 OAuth 平台集成
3. 开放给公众的完整开发者平台

## 6. 当前实现结果
1. 已建立 `SyncSourceRecord` 与 `SyncRunRecord`
2. 已落地 `/admin/sync`
3. 已建立同步边界与手动触发接口骨架
4. 已在 MySQL 中初始化 `sync_sources` 与 `sync_runs`
5. 已预置 `notes-import`、`project-catalog`、`search-runtime` 三个同步源
