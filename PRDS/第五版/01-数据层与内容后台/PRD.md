# 第五版 - 数据层与内容后台 PRD
## 1. 功能定位
第五版继续保留第四版的数据层与内容后台边界，但目标从“骨架搭建”转为“稳定运行与部署收尾”。

## 2. 当前版本状态
状态：已完成

## 3. 当前实现范围
1. 统一后台内容读取
2. 统一后台内容详情与治理视图
3. 明确博客、笔记、项目内容源目录
4. 为部署文档提供内容目录规则

## 4. 本轮必须完成
1. `/admin`、`/admin/content`、`/admin/content/:id` 保持可访问
2. 文档中明确：
   - 博客内容放在 `content/blog/*.md`
   - 笔记内容放在 `content/notes/*.md`
   - 项目内容放在 `content/projects/*.md`

## 5. 暂不要求的内容
1. 后台在线编辑
2. 数据库写内容
3. 审核流与发布流
4. 用户权限系统
