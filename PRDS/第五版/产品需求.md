# 第五版产品需求
## 1. 第五版定位
第五版是这个项目从“第四版后端骨架”收束到“`v1.0` 可部署个人博客”的最终收尾版。

这一版不再继续扩张平台能力，重点是把第四版已经进入代码结构的后端能力真正跑通，并整理成可以部署到服务器的稳定版本。

第五版要解决的问题是：
- 第四版虽然已经有后台页、MySQL 表结构和 Redis 运行态边界，但本地与部署链路还没有完全闭环
- `/admin/search`、`/admin/stats`、`/admin/tasks`、`/admin/sync` 还需要完成真实运行验证
- 搜索重建、任务记录、同步记录还需要留下正式持久化结果
- 本地启动方式、服务器部署方式、环境变量和内容放置规则还没有整理成 `v1.0` 文档

## 2. 第五版总体策略
第五版采用“收尾优先、部署优先、闭环优先”的策略：

1. 先修复并确认服务端真实运行问题
2. 先跑通 MySQL 与 Redis 的真实链路
3. 先让后台页展示真实状态和真实记录
4. 先整理部署配置、运行命令与环境变量
5. 保持个人博客定位，不新增第五版之外的新平台能力

## 3. 第五版目标
第五版需要完成以下目标：

1. 在第四版骨架基础上完成真实运行闭环
2. 完成 MySQL 与 Redis 的真实连接与数据写入验证
3. 完成搜索索引真实重建与后台验证
4. 完成统计、任务、同步模块的最小可用闭环
5. 完成本地运行与服务器部署所需配置
6. 完成 `v1.0` 收尾文档

当前状态：已完成

## 4. 第五版范围
### 4.1 本轮必须完成的上线能力

1. 服务端可稳定启动并提供 SSR 页面与后台接口
2. `BLOG_DATABASE_URL`、`BLOG_REDIS_URL`、`SITE_URL` 等关键环境变量有正式配置说明
3. `/admin/search` 可完成真实搜索重建，并写入：
   - `search_documents`
   - `search_rebuild_runs`
4. `/admin/stats` 可生成并展示真实统计快照，并写入：
   - `stats_snapshots`
5. `/admin/tasks` 可展示真实任务执行记录，并写入：
   - `task_runs`
6. `/admin/sync` 可完成最小可用同步闭环，并写入：
   - `sync_sources`
   - `sync_runs`
7. 本地开发运行命令、生产构建命令、部署步骤、反向代理要求和运行目录约定有正式文档
8. 明确博客、笔记、项目内容后续分别放到哪里

### 4.2 本轮明确不进入 v1.0 的内容

1. 完整用户系统
2. 多角色权限与复杂 RBAC
3. 评论系统与社区能力
4. 对公众开放的平台化 API
5. 复杂工作流编排
6. 独立 worker、消息队列与分布式调度系统
7. 新增第五版之外的平台型内容模块
8. 大规模视觉重设计

## 5. 模块范围说明
### 5.1 数据层与内容后台

第五版继续沿用第四版“仓库内 Markdown 作为内容源”的方案，不切换到数据库写内容。

本轮要求：
- 后台读模型继续统一读取 `blog / notes / projects`
- 后台作为治理和验证入口继续保留
- 明确内容放置规则：
  - 博客内容：`content/blog/*.md`
  - 笔记内容：`content/notes/*.md`
  - 项目内容：`content/projects/*.md`

### 5.2 搜索与索引基础设施

第五版要求搜索能力从“骨架可用”进入“真实可验证”：

- 真实执行搜索重建
- 搜索文档正式写入 MySQL
- Redis 正式记录重建运行态
- `/admin/search` 能显示真实状态、记录和覆盖结果

### 5.3 统计与任务系统

第五版要求：

- `/admin/stats` 每次读取能生成和更新最小统计快照
- `/admin/tasks` 能看到搜索重建、同步运行等真实任务记录
- 统计与任务以治理用途为主，不扩展成 BI 系统或调度中心

### 5.4 外部同步与开放能力

第五版只要求最小可用闭环：

- `/admin/sync` 可展示同步源
- 手动触发同步可生成正式运行记录
- 当前同步仍允许保持“站内导入器 / 占位同步器”级别，不接第三方 OAuth 平台

## 6. 部署范围
第五版必须明确以下部署边界：

1. 运行进程：Rust SSR 服务
2. 数据库：MySQL
3. 缓存 / 运行态：Redis
4. 反向代理：Nginx 或同类代理
5. 内容目录：随仓库部署
6. 构建产物目录与启动命令

## 7. 成功标准
第五版完成后，应满足以下标准：

1. 项目达到 `v1.0 可部署` 标准
2. 本地可按文档稳定启动
3. 服务端后台页能展示真实 MySQL / Redis 状态
4. 搜索重建、统计快照、任务记录、同步记录至少完成一轮真实写入验证
5. 部署文档足够支持用户自行准备服务器环境并上线
6. 文档中明确说明哪些内容被刻意留在 `v1.0` 之外
