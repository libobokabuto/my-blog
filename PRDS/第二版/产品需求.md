# 第二版产品需求

## 1. 第二版定位

第二版不再解决“站点有没有正式跑起来”的问题，而是解决“这个站点是否更好浏览、更好发现内容、更像一个完整产品”的问题。

第一版已经完成了正式骨架、核心页面和基础站点输出。第二版应该在此基础上，让这个网站从“可用”升级到“更完整、更好读、更好找”。

## 2. 第二版总体策略

第二版建议采用：

- 以现有产品打磨为主
- 配合少量高价值新增功能

不建议第二版一上来就引入评论、数据库、后台、用户系统这类高复杂度能力，因为这会明显打断当前内容站和 Rust 学习项目的推进节奏。

## 3. 为什么第二版不建议纯打磨

如果第二版只做样式微调和文案优化，版本价值不够明显。

如果第二版直接做评论、后台、数据库，复杂度又会陡增。

所以第二版更合理的方向是：

1. 打磨现有内容体验
2. 补齐内容系统缺的关键链路
3. 新增一个真正有使用价值、但复杂度可控的能力

## 4. 第二版目标

第二版需要完成以下目标：

1. 让 `notes` 成为可完整浏览的内容系统
2. 让 `projects` 成为可完整浏览的展示系统
3. 提升内容发现能力
4. 提升标签、归档和页面间跳转体验
5. 继续完善 SEO 与公开访问质量

## 5. 第二版建议范围

### 5.1 必做范围

1. 笔记详情页 `/notes/:slug`
2. 项目详情页 `/projects/:slug`
3. 标签归档页 `/tags/:tag`
4. 站内搜索页 `/search`
5. 更完整的 SEO 信息
6. 继续优化首页、博客、笔记、项目页的内容结构

### 5.2 建议增强

1. 搜索结果高亮
2. 首页加入更明确的内容入口节奏
3. 内容列表加入更清晰的筛选/归档提示
4. 项目页增加状态分组

## 6. 第二版不做

第二版仍然明确不做以下内容：

1. 评论系统
2. 登录注册
3. 用户系统
4. 数据库
5. 后台管理
6. 富文本编辑器
7. 多角色权限

## 7. 第二版核心判断

第二版最适合做的是：

- 让已经存在的内容真正变得完整
- 增强内容发现能力
- 提高站点作为“长期公开内容档案”的质量

而不是转向做复杂平台型系统。

## 8. 页面与功能范围

第二版正式实现的重点页面与能力建议为：

1. `/notes/:slug`
2. `/projects/:slug`
3. `/tags/:tag`
4. `/search`

并对以下已有页面做结构升级：

1. `/`
2. `/blog`
3. `/notes`
4. `/projects`

## 9. 内容模型方向

第二版不改变“blog / notes / projects”三类内容模型，但会增强它们的浏览方式：

### `blog`

- 继续作为正式长文系统
- 增强标签与归档体验

### `notes`

- 从“只有列表”升级到“列表 + 详情”
- 更强调学习过程和实验记录

### `projects`

- 从“只有列表”升级到“列表 + 详情”
- 更强调项目背景、过程和结果展示

## 10. 成功标准

第二版完成后，应满足以下标准：

1. 站内搜索可用
2. 标签归档可访问
3. `notes` 和 `projects` 都具备完整内容链路
4. 页面间的内容跳转更自然
5. SEO 与公开访问质量高于第一版

## 11. 当前状态

状态：已完成

第二版核心功能与收尾优化已经完成。

本轮重点不再是新增大功能，而是继续完成：

- 页面级 SEO 完整度补齐
- 正式发布地址配置收口
- 关键页面公开访问质量提升
