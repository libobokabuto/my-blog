# 第二版开发计划

## 1. 开发目标

在第一版正式站点已经完成的基础上，完成第二版的内容体验升级与内容发现能力建设。

## 2. 开发策略

第二版采用以下策略推进：

1. 先补完整内容链路，再做新增功能
2. 先做可控的检索与归档能力，不引入数据库
3. 保持静态内容源 + Rust SSR 的主路线不变
4. 优先做“用户一眼能感知到价值”的升级

## 3. 阶段划分

### 阶段一：内容链路补齐

目标：

- 为 `notes` 增加详情页
- 为 `projects` 增加详情页
- 补齐相关内容读取逻辑

状态：已完成

当前推进：

- `notes` 详情页 `/notes/:slug` 已完成
- 已补齐笔记详情路由、正文读取与列表到详情跳转
- `projects` 详情页 `/projects/:slug` 已完成

### 阶段二：内容发现能力

目标：

- 实现标签归档页
- 实现站内搜索页
- 明确搜索索引与匹配策略

状态：已完成

当前推进：

- 标签归档页 `/tags/:tag` 已完成
- 站内搜索页 `/search` 已完成
- 已补齐标签入口、搜索索引与页面跳转链路

### 阶段三：页面体验升级

目标：

- 优化首页内容入口结构
- 优化博客、笔记、项目列表页的信息层级
- 提升详情页阅读和跳转体验

状态：已完成

当前推进：

- 已同步优化首页、博客、笔记、项目页的内容入口和跳转体验
- 已补齐搜索结果高亮、项目列表状态分组与归档提示

### 阶段四：公开访问质量提升

目标：

- 继续完善 SEO 元信息
- 继续完善站点公开访问质量
- 视需要补充部署和运营相关说明

状态：已完成

当前推进：

- 已补齐新增详情页、标签页、搜索页的页面级 SEO 信息
- 已增强站点 sitemap、页面描述与公开访问时的可读性

## 4. 当前建议优先级

建议第二版按以下顺序推进：

1. `notes` 详情页
2. `projects` 详情页
3. 标签归档页
4. 站内搜索
5. 页面体验与 SEO 打磨

## 5. 第二版完成判断

第二版可以认为完成，当以下条件同时满足：

1. `notes` 与 `projects` 都具备列表 + 详情链路
2. 搜索页与标签归档页可用
3. 首页、博客、笔记、项目页体验明显优于第一版
4. 第二版未引入超出当前节奏的大型系统复杂度
