# 第二版收尾优化

## 1. 文档定位

这份文档用于记录第二版在“核心需求已完成”之后，仍需要继续收口的优化项。

它不是第三版需求，而是第二版上线质量与完成度收尾文档。

## 2. 当前状态

状态：已完成

第二版核心功能已完成：

- `/notes/:slug`
- `/projects/:slug`
- `/tags/:tag`
- `/search`

当前仍需继续收口的重点在于：

- SEO 元信息完整度
- 发布域名与公开访问配置
- 搜索结果质量
- 关键页面稳定性验证

## 2.1 本轮已完成

- 已抽离正式站点 `SITE_URL` 配置
- 已为主要页面补齐 canonical 与公开分享元信息
- 已确保 `rss.xml` 与 `sitemap.xml` 使用生产可配置地址
- 已完成 `cargo check` 与基于 `x86_64-pc-windows-msvc` 的测试验证

## 3. 本轮收尾目标

本轮优先完成以下内容：

1. 抽离正式站点 `site_url` 配置
2. 为主要页面补齐更完整的页面级 SEO 信息
3. 确保 `rss.xml` 与 `sitemap.xml` 使用正式站点地址
4. 为关键页面补基本稳定性验证

## 4. 本轮范围

### 4.1 必做

1. `site_url` 从 `site_addr` 中解耦
2. 首页、博客、笔记、项目、详情页、标签页补 `canonical`
3. 主要公开页面补 `og:title`、`og:description`、`og:type`
4. 搜索页保持 `noindex`，但补齐 canonical 与公开访问边界说明
5. 为 sitemap / rss 生成使用生产可配置地址

### 4.2 建议增强

1. 搜索结果排序优化
2. 搜索结果命中提示优化
3. 内容路由与 hydration 回归验证

## 5. 暂不要求

以下内容不放在第二版收尾中：

1. 评论系统
2. 用户系统
3. 数据库
4. 后台
5. 搜索筛选器与复杂检索面板

## 6. 完成标准

第二版收尾完成后，应满足以下标准：

1. sitemap 与 rss 在生产环境使用正确绝对地址
2. 主要页面元信息完整度高于当前状态
3. 关键公开页面具备更稳定的分享与收录基础
4. 第二版可以作为“完成核心功能并完成上线收口”的阶段结束
