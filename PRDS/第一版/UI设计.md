# 第一版 UI 设计

## 1. 设计目标

第一版 UI 不追求“先随便做一个能用的壳子”，而是追求：

1. 一眼看上去就是一个认真制作的个人网站
2. 页面有统一风格
3. 内容阅读舒服
4. 后续版本可以在现有基础上持续打磨

因此，第一版 UI 的策略不是“后面推倒重来”，而是“先定好正确的视觉方向，再逐步精修”。

## 2. 整体设计原则

1. 以内容阅读体验为核心
2. 首页要有个人辨识度
3. 页面结构清晰，信息层级明确
4. 视觉风格统一，避免不同页面像不同网站
5. 动效克制，只做必要的强调

## 3. 视觉方向

第一版建议采用：

- 干净、现代、偏内容型的设计
- 强调排版、留白、层次
- 不走花哨炫技路线
- 不使用模板味很重的默认布局

建议视觉关键词：

- 稳定
- 清晰
- 克制
- 有个人气质
- 可长期演进

## 4. 页面体验要求

### 首页

首页需要承担“第一印象”职责，应包含：

- 清晰的个人身份介绍
- 简洁但有记忆点的首屏
- 最近更新内容入口
- 项目展示入口

### 博客列表页

要求：

- 易于扫描
- 标题突出
- 摘要和时间信息清楚
- 标签展示不过度抢眼

### 博客详情页

要求：

- 阅读体验优先
- 字号、行高、段落间距舒适
- 代码块清晰
- 标题层级明显

### 笔记页

要求：

- 与博客有相同家族风格
- 但视觉上可以更轻、更简洁

### 项目页

要求：

- 卡片化展示项目
- 明确项目名称、简介、技术栈、链接

### 关于页

要求：

- 更偏个人表达
- 不是简历翻版

## 5. 第一版 UI 结论

第一版需要做出“已经可以公开展示”的页面质量，而不是临时占位稿。

原因：

1. 后续如果推倒重做，成本高
2. 第一印象会影响你对项目的持续投入意愿
3. 这个网站本身也是你的作品展示

所以第一版的 UI 目标应是：

- 不是最终极版本
- 但绝不是功能壳子
- 要能成为后续持续打磨的正确基底

## 6. 后续迭代方向

后续 UI 迭代应聚焦于：

1. 细节微调
2. 动效增强
3. 版式优化
4. 响应式体验打磨
5. 品牌感和个人气质强化

而不是重做整个视觉体系。
