# 第一版产品需求

## 1. 产品定位

第一版个人网站定位为一个以博客为核心的个人内容站点，用于：

- 展示个人信息
- 发布博客文章
- 记录学习笔记
- 展示个人项目

同时，这个项目也是一个 Rust 实战学习项目。当前技术方向已调整为全栈 Rust。

## 2. 第一版目标

第一版需要完成以下目标：

1. 搭建完整的网站基础结构
2. 使用全栈 Rust 建立网站的正式实现方向
3. 支持博客内容发布与展示
4. 形成统一、清晰、可信的视觉风格
5. 为后续笔记、项目展示和更多交互功能打下基础

## 3. 技术路线调整

第一版正式技术路线调整为：

- 前端：`Leptos`
- 后端：`Leptos SSR`
- 样式：CSS
- 内容：Markdown

这意味着第一版不再采用“Rust 后端 + 原生前端模板”的正式实现路线，而是采用“前后端都以 Rust 为核心”的实现路线。

## 4. 页面范围

为了控制全栈 Rust 第一版复杂度，第一版最初的正式实现范围先收缩为：

1. 首页 `/`
2. 博客列表页 `/blog`
3. 博客详情页 `/blog/:slug`
4. 关于页 `/about`

以下页面保留在产品规划中，但不作为第一批正式实现目标：

1. 笔记列表页 `/notes`
2. 项目展示页 `/projects`

当前状态更新：

- 第一版正式页面现已扩展为 `/`、`/blog`、`/blog/:slug`、`/notes`、`/projects`、`/about`
- `notes` 和 `projects` 已从“规划保留”进入“正式实现完成”

## 5. 功能范围

### 5.1 第一批必做功能

1. 全站导航
2. 首页展示
3. 博客列表展示
4. 博客详情展示
5. Markdown 内容渲染
6. 标签展示
7. 文章按时间排序
8. 关于页展示

### 5.2 第一版可选增强

1. 上一篇 / 下一篇文章跳转
2. 基础 SEO 信息
3. `rss.xml`
4. `sitemap.xml`

当前状态更新：

- 上一篇 / 下一篇导航已完成
- 基础 SEO、`rss.xml`、`sitemap.xml` 已完成第一版基础接入

## 6. 第一版不做

为控制复杂度，第一版明确不做以下内容：

1. 登录注册
2. 后台管理系统
3. 评论系统
4. 站内搜索
5. 用户系统
6. 数据库
7. 富文本编辑器
8. 多角色权限
9. 高频复杂交互

## 7. 内容模型

第一版内容仍分为三类：

1. `blog`
   正式发布的完整文章
2. `notes`
   学习记录、短内容、实验记录
3. `projects`
   项目展示内容

第一版当前已完成三类内容的基础接入：

- `blog` 已具备列表、详情和阅读连续性
- `notes` 已具备列表展示
- `projects` 已具备列表展示

## 8. 现有 Demo 的定位

当前已经完成的前端 demo 不作废。

当前 demo 的定位是：

- 视觉参考稿
- 信息架构参考稿
- 页面气质参考稿

它不是未来正式实现的技术基础代码，而是 Leptos 正式实现时的视觉与结构参考。

## 9. 目录职责调整

### `frontend/`

`frontend/` 目录中的现有内容视为旧的前端原型实现，不作为最终技术路线继续扩展。

后续如果保留该目录，其职责仅限于：

- 保存旧 demo 或过渡原型
- 作为视觉实现参考

### `static/`

`static/` 目录用于存放静态资源与视觉参考稿，例如：

- 图片
- 图标
- 字体
- demo HTML/CSS/JS

后续不会把 `static/` 当作正式前端项目根目录来持续开发。

## 10. 成功标准

第一版完成后，应满足以下标准：

1. 全栈 Rust 项目骨架可运行
2. 首页、博客列表、博客详情、笔记页、项目页、关于页可访问
3. Markdown 内容可以正常展示
4. 页面结构清晰、视觉统一
5. 基础 RSS 与 sitemap 可访问
6. 后续可以在当前全栈 Rust 架构上继续迭代
