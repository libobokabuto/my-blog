# 第一版产品需求

## 1. 产品定位

第一版个人网站定位为一个以博客为核心的个人内容站点，用于：

- 展示个人信息
- 发布博客文章
- 记录学习笔记
- 展示个人项目

第一版目标不是做一个功能庞杂的平台，而是先做出一个可公开访问、可持续更新、可长期迭代的个人网站基础版本。

## 2. 第一版目标

第一版需要完成以下目标：

1. 搭建完整的网站基础结构
2. 支持博客内容发布与展示
3. 支持笔记内容展示
4. 支持项目展示
5. 支持个人介绍展示
6. 形成统一、清晰、可信的视觉风格

## 3. 目标用户

第一版的主要用户包括：

- 网站作者本人
- 访问作者博客的朋友、同学、同行
- 通过项目或文章认识作者的潜在访客

## 4. 页面范围

第一版包含以下页面：

1. 首页 `/`
2. 博客列表页 `/blog`
3. 博客详情页 `/blog/:slug`
4. 笔记列表页 `/notes`
5. 项目展示页 `/projects`
6. 关于页 `/about`

## 5. 功能范围

### 5.1 必做功能

1. 全站导航
2. 博客列表展示
3. 博客详情展示
4. Markdown 内容渲染
5. 笔记列表展示
6. 项目列表展示
7. 首页聚合展示最新内容
8. 标签展示
9. 文章按时间排序

### 5.2 推荐纳入第一版的功能

1. 上一篇 / 下一篇文章跳转
2. 基础 SEO 信息
3. `rss.xml`
4. `sitemap.xml`

## 6. 第一版不做

为控制复杂度，第一版明确不做以下内容：

1. 登录注册
2. 后台管理系统
3. 评论系统
4. 站内搜索
5. 用户系统
6. 数据库
7. 富文本编辑器
8. 前后端分离接口体系
9. 多角色权限

## 7. 内容模型

第一版内容分为三类：

1. `blog`
   正式发布的完整文章
2. `notes`
   学习记录、短内容、实验记录
3. `projects`
   项目展示内容

## 8. 成功标准

第一版完成后，应满足以下标准：

1. 访客可以顺畅访问主要页面
2. 博客内容可以正常展示
3. 页面结构清晰、视觉统一
4. 网站已经具备“可长期打磨”的基础外观
5. 后续迭代可以在当前页面体系上持续优化，而不是整体推翻重做

## 9. 关于第一版视觉目标的结论

第一版不应该只是“功能占位页”，也不建议以后推倒重来。

第一版更合理的目标是：

- 做出一个已经具备审美基础的前端版本
- 保证结构、气质、信息层级一开始就是对的
- 后续主要是在现有基础上持续打磨，而不是重做整站视觉

也就是说，第一版不要求把每个像素都打磨到极致，但必须做到：

1. 能公开展示，不廉价
2. 风格统一，不拼凑
3. 有辨识度，不像默认模板
4. 后续可渐进优化
