# 首页 PRD

## 功能定位

首页是第一版网站的总入口，负责同时完成三件事：

- 说明这个站点是什么
- 给出当前最值得阅读的内容入口
- 解释这个项目为什么存在、现在推进到哪一步

它不是简历页，也不是纯导航页，而是一张“公开工作现场”的首页。

## 第一版状态

状态：已在第一批正式实现中完成

说明：

- 当前正式项目已经实现 `/`
- 首页已经接入 Leptos SSR 正式项目
- 首页中“最新博客”已接入真实 `blog` 内容
- 首页中的“最近笔记”和“代表项目”已经改为读取真实 `notes`、`projects` 内容

## 当前实现范围

第一版首页当前已包含：

1. 顶部导航
2. 首屏身份说明
3. 当前主题与第一阶段目标卡片
4. 最新博客入口区
5. 最近笔记预告区
6. 代表项目预告区
7. 第一版重点与当前边界说明区

## 首页应传达的信息

访客进入首页后，应能快速判断：

1. 这是一个以博客为核心的个人内容站
2. 当前站点围绕 Rust 学习和长期项目推进展开
3. 站点已经进入正式的 Leptos SSR 实现阶段
4. 下一步最自然的阅读路径是博客、笔记或项目页

## 第一版暂不要求

- 简历式经历堆叠
- 搜索框
- 评论、订阅、登录入口
- 炫技型动效
- 虚假数据展示
