# 第一版开发计划

## 1. 开发目标

基于第一版产品需求与 UI 方向，完成一个可上线、可展示、可持续迭代的 Rust 个人博客网站基础版本。

## 2. 开发策略

第一版开发策略如下：

1. 先完成完整结构
2. 再完成核心页面
3. 再统一前端风格
4. 最后补齐内容展示细节

第一版的开发目标不是“最低可运行”，而是“基础版本就具备正式展示价值”。

## 3. 阶段划分

### 阶段一：项目基础搭建

目标：

- 初始化 Rust 项目
- 建立源码目录结构
- 建立模板目录
- 建立静态资源目录
- 建立内容目录
- 建立 PRDS 和 LEARNING 目录

交付结果：

- 项目骨架可用
- 目录结构稳定

### 阶段二：前端 Demo 确认

目标：

- 先做首页、博客列表、博客详情等页面的前端展示稿
- 确认视觉方向是否符合预期
- 确认整体风格是否可延续

交付结果：

- 可预览的前端 Demo
- 视觉方向定稿

### 阶段三：核心页面实现

目标：

- 实现首页
- 实现博客列表页
- 实现博客详情页
- 实现笔记页
- 实现项目页
- 实现关于页

交付结果：

- 第一版页面全部可访问

### 阶段四：内容系统接入

目标：

- 接入 Markdown 内容
- 解析 front matter
- 实现博客、笔记、项目内容渲染
- 实现排序和标签展示

交付结果：

- 网站具备真实内容展示能力

### 阶段五：发布前完善

目标：

- 补齐基础 SEO
- 生成 RSS
- 生成 sitemap
- 完善响应式体验
- 做基础测试和自检

交付结果：

- 第一版具备上线准备

## 4. 当前结论

第一版前端开发要求如下：

1. 第一版页面必须认真做
2. 第一版要具备正式展示价值
3. 后续在现有设计基础上迭代优化
4. 不以“后面推倒重来”为预期

## 5. 当前下一步

当前最合适的下一步是：

1. 先根据本文件和 UI 文档制作前端 Demo
2. 看页面气质、信息布局、视觉风格是否符合预期
3. 再决定进入正式页面实现
