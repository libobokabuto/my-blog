# 第一版开发计划

## 1. 开发目标

基于第一版产品需求与视觉方向，完成一个可运行、可继续扩展，并以 `Leptos + Leptos SSR + Markdown` 为正式技术路线的个人网站第一版。

第一版的重点不是一次性把所有功能都做完，而是先把博客主链路和正式项目骨架打通。

## 2. 开发策略

第一版采用以下策略推进：

1. 保留现有 demo，继续作为视觉参考稿使用
2. 正式实现全面切换到全栈 Rust 路线
3. 收缩第一批正式页面范围，避免过早摊大
4. 优先打通“内容源 -> 路由 -> 页面渲染 -> 站点结构”主链路
5. 将展示文档、产品规划、版本记录分开维护

## 3. 第一版阶段划分

### 阶段一：正式项目骨架

目标：

- 初始化 `Leptos SSR` 项目
- 明确 `app / client / server / style` 正式结构
- 建立 Rust workspace
- 明确 `static` 的参考稿职责

状态：已完成

交付结果：

- 正式 Leptos SSR 项目骨架可运行

### 阶段二：视觉参考迁移

目标：

- 以 `static/demo-v1` 为视觉参考
- 在正式项目中搭出首页、博客、关于页的页面壳子
- 保持 demo 可继续作为后续迭代参考

状态：已完成

交付结果：

- 正式项目中的首页和博客页已经具备接近 demo 的组件结构与视觉方向

### 阶段三：第一批正式页面

目标：

- 实现首页 `/`
- 实现博客列表页 `/blog`
- 实现博客详情页 `/blog/:slug`
- 实现关于页 `/about`

状态：已完成

交付结果：

- 第一批正式页面全部可访问

### 阶段四：博客内容系统接入

目标：

- 接入本地 `Markdown` 内容
- 解析 front matter
- 实现博客列表和详情渲染
- 支持文章排序与基础标签展示

状态：已完成

交付结果：

- 网站已具备真实博客内容展示能力

### 阶段五：第一版待补项

目标：

- 正式接入 `notes`
- 正式接入 `projects`
- 继续细化内容组织与页面体验

状态：已完成

说明：

本阶段已完成 `notes` 与 `projects` 的正式接入。

### 阶段六：上线前完善

目标：

- 补基础 SEO
- 生成 RSS
- 生成 sitemap
- 完善响应式体验
- 做基础测试和上线前自检

状态：已完成第一轮基础接入

## 4. 当前完成情况

当前已经完成的是“第一版第一批正式实现范围”，包括：

1. 全栈 Rust 正式项目骨架
2. 首页、博客列表、博客详情、笔记页、项目页、关于页
3. 基于 Markdown 的博客、笔记、项目内容系统
4. 博客详情页上一篇 / 下一篇导航
5. 基础 SEO、RSS、sitemap
6. demo 参考稿到正式项目组件结构的第一次迁移
7. 删除旧 `frontend/` 原型，清理历史残留目录

## 5. 第一版剩余工作怎么定义

第一版剩余内容建议拆成两类：

### 5.1 第一版待继续打磨

- 内容结构与页面体验继续打磨
- 更完整的标签归档体验
- SEO 信息继续细化

## 6. 不建议直接并入第二版的内容

以下内容虽然本轮没做，但更适合作为第一版后续迭代，而不是立即定义成第二版：

- 内容体验的进一步打磨
- 标签归档体验增强

原因是这些内容仍然直接服务于当前站点的第一版完整度，不属于新的产品方向。

## 7. 第二版更适合承接什么

第二版更适合放真正的新能力，例如：

- 站内搜索
- 评论系统
- 数据库存储
- 后台管理
- 更复杂的内容管理和交互系统

## 8. 当前下一步建议

当前最合适的推进顺序是：

1. 继续完善 `notes` 的内容组织
2. 继续完善 `projects` 的展示细节
3. 继续细化博客阅读体验与 SEO
4. 准备部署与上线说明
