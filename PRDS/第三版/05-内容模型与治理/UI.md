# 内容模型与治理 UI

## 当前状态

状态：已在第一批正式实现中完成

## 视觉方向

字段增强后的内容页应更像“完整档案”，但不应变成后台元数据面板。

## 页面气质

- 元信息更完整
- 仍以正文为主
- 校验能力主要体现在流程，不在页面上过度暴露

## 建议结构

1. 标题区补充系列、阅读时长、阶段、来源等上下文
2. 项目页展示背景、角色、时间线、结果、复盘
3. 系列页承担跨文章组织职责

## 后续优化方向

- 后续可继续把主题页与系列页统一
- 第四版若有后台，再考虑可视化治理面板

## 当前实现结果

- 博客详情已补充系列与阅读时长
- 笔记详情已补充阶段、来源、实验状态
- 项目详情已补充背景、角色、时间线、结果与复盘
