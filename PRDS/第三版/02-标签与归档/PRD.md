# 标签与归档 PRD

## 功能定位

这个模块用于在第三版中把第二版已经可用的标签归档能力，升级成更完整的内容组织入口。

它不再只是单个标签结果页，而是：

- 提供 `/tags` 标签总览页
- 提供 `/archive` 归档索引页
- 让 `blog / notes` 形成更清晰的按主题、按时间浏览路径

## 当前状态

状态：已在第一批正式实现中完成

## 第三版目标

- 在保留 `/tags/:tag` 的基础上新增 `/tags`
- 新增 `/archive` 作为正式归档入口
- 让首页、个人主页、博客、笔记、搜索都能自然进入这两类索引页

## 当前实现范围

本轮优先实现：

1. 标签总览页 `/tags`
2. 归档索引页 `/archive`
3. 基于现有 Markdown 内容的标签聚合与时间聚合
4. 与首页、`/me`、`blog`、`notes`、`search` 的导航打通

## 本轮补充约束

- 不新增数据库
- 不做后台式多维筛选器
- 项目内容暂不进入时间归档主列表，等待第三版内容模型增强时再统一处理

## 暂不要求

- 系列页
- 主题后台管理
- 可配置权重排序
- 基于访问数据的热门标签统计

## 当前实现结果

- `/tags` 已正式接入站点
- `/archive` 已正式接入站点
- `/tags/:tag` 与总览、归档、搜索已形成跳转关系
- 当前归档主轴仅覆盖博客与笔记，项目内容保留到第三版内容模型增强时再统一处理
