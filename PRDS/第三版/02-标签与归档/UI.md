# 标签与归档 UI

## 当前状态

状态：已在第一批正式实现中完成

## 视觉方向

这一轮的标签与归档页应更像“内容索引页”而不是“筛选工具页”。

应保持：

- 编辑部式内容组织感
- 清楚的分区与层次
- 快速扫描与继续跳转能力

## 页面气质

标签总览页建议更像主题目录，归档页建议更像时间轴索引。

其中：

- `/tags` 更强调主题密度与入口分发
- `/archive` 更强调时间结构与更新节奏
- `/tags/:tag` 继续承担单主题结果页职责

## 建议模块层次

1. 顶部：页面定位与总量摘要
2. 中部：标签卡片或年度归档分组
3. 下部：继续浏览入口，如搜索、博客、笔记、个人主页

## 后续优化方向

- 后续可在第三版内容模型增强后再决定是否把项目纳入归档主轴
- 搜索升级时可复用标签与归档页的索引表达
- 系列页或主题页适合在这套结构稳定后再引入

## 当前实现结果

- `/tags` 已按主题目录式入口落地
- `/archive` 已按年份索引式结构落地
- `/tags/:tag` 与总览页、归档页之间的视觉层级已经拉开
