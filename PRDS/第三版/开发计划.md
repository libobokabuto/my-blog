# 第三版开发计划

## 1. 当前状态

状态：已完成

当前第三版五个阶段都已完成：个人主页、内容组织、内容关联、搜索升级、内容治理已经全部接入正式实现。

## 2. 开发目标

在第二版已完成内容链路补齐的基础上，推进内容组织、关联、检索和治理能力升级。

## 3. 开发策略

第三版建议采用以下策略推进：

1. 先做个人主页与内容组织骨架，再做关联与体验增强
2. 先做静态内容可控增强，不引入数据库
3. 先做长期维护收益高的能力，再考虑展示层扩展
4. 保持 Rust SSR + Markdown 主路径不变

## 4. 阶段划分

### 阶段一：个人主页与站点枢纽升级

目标：

- 增加个人主页或新的公开工作台页面
- 让个人主页连接内容、项目、主题、最近动态
- 明确主页与 about 页的分工

状态：已完成

当前子任务：

1. 已完成首页、`/me`、`/about` 的职责边界同步
2. 已完成个人主页结构、内容入口、最近动态与轻量统计
3. 已完成 `blog / notes / projects` 的统一主页聚合接入
4. 已预留后续搜索升级与内容关联可复用的服务端聚合接口

### 阶段二：内容组织升级

目标：

- 增加标签总览、归档索引或主题页
- 统一内容索引入口
- 明确内容类型之间的组织方式

状态：已完成

当前子任务：

1. 已完成 `/tags` 标签总览页
2. 已完成 `/archive` 归档索引页
3. 已完成标签与时间聚合结构
4. 已完成与首页、个人主页、博客、笔记、搜索的导航关系打通

### 阶段三：内容关联升级

目标：

- 为 `blog / notes / projects` 增加相关文章或相关内容
- 让详情页之间形成阅读网络

状态：已完成

当前子任务：

1. 已完成三类内容的关联计算结构
2. 已完成 `blog / notes / projects` 详情页的关联阅读
3. 已完成系列、标签、技术栈和主题重叠的静态关联

### 阶段四：搜索升级

目标：

- 优化相关度排序
- 增加类型或标签筛选
- 提升结果摘要与命中提示质量

状态：已完成

当前子任务：

1. 已完成静态搜索相关度排序优化
2. 已完成类型筛选与标签筛选
3. 已完成结果命中解释与上下文表达优化

### 阶段五：内容治理与发布质量

目标：

- 增加 front matter 校验
- 增加重复 slug、坏链接、缺失字段检查
- 补齐部署与发布相关文档

状态：已完成

当前子任务：

1. 已完成 `blog / notes / projects` front matter 字段扩展
2. 已完成系列页 `/series/:slug`
3. 已完成内容校验脚本，覆盖 front matter、slug、坏链接和关键缺失项

## 5. 建议优先顺序

建议第三版按以下顺序推进：

1. 个人主页
2. 标签总览 / 归档页
3. 内容关联模块
4. 搜索升级
5. 内容模型字段增强
6. 内容校验与发布治理

## 6. 完成判断

第三版可以认为完成，当以下条件同时满足：

1. 个人主页已成为站点公开导航枢纽
2. 内容组织入口明显多于第二版
3. 内容详情页已具备稳定的关联阅读能力
4. 搜索结果质量与筛选能力明显优于第二版
5. 新增内容的校验与维护流程更稳定
