# 第三版开发计划

## 1. 当前状态

状态：未开始

当前仅完成第三版 PRDS 初稿，尚未进入正式实现。

## 2. 开发目标

在第二版已完成内容链路补齐的基础上，推进内容组织、关联、检索和治理能力升级。

## 3. 开发策略

第三版建议采用以下策略推进：

1. 先做内容组织与关联，再做体验增强
2. 先做静态内容可控增强，不引入数据库
3. 先做长期维护收益高的能力，再考虑展示层扩展
4. 保持 Rust SSR + Markdown 主路径不变

## 4. 阶段划分

### 阶段一：内容组织升级

目标：

- 增加标签总览、归档索引或主题页
- 统一内容索引入口
- 明确内容类型之间的组织方式

状态：未开始

### 阶段二：内容关联升级

目标：

- 为 `blog / notes / projects` 增加相关文章或相关内容
- 让详情页之间形成阅读网络

状态：未开始

### 阶段三：搜索升级

目标：

- 优化相关度排序
- 增加类型或标签筛选
- 提升结果摘要与命中提示质量

状态：未开始

### 阶段四：内容治理与发布质量

目标：

- 增加 front matter 校验
- 增加重复 slug、坏链接、缺失字段检查
- 补齐部署与发布相关文档

状态：未开始

## 5. 建议优先顺序

建议第三版按以下顺序推进：

1. 标签总览 / 归档页
2. 内容关联模块
3. 搜索升级
4. 内容模型字段增强
5. 内容校验与发布治理

## 6. 完成判断

第三版可以认为完成，当以下条件同时满足：

1. 内容组织入口明显多于第二版
2. 内容详情页已具备稳定的关联阅读能力
3. 搜索结果质量与筛选能力明显优于第二版
4. 新增内容的校验与维护流程更稳定
