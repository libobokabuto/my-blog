# 个人主页 UI

## 当前状态

状态：已在第一批正式实现中完成

## 视觉方向

个人主页应更像“公开工作台”而不是“监控面板”。

应保持：

- 内容站气质
- 编辑感与长期项目感
- 清晰分区和较强的个人辨识度

## 页面气质

主页建议同时具备：

- 个人介绍页的温度
- 内容导航页的清晰度
- 项目现场的进行中感觉

## 建议模块层次

1. 顶部：身份与当前状态
2. 中部：内容入口、最近动态与重点项目
3. 下部：主题入口与轻量统计
4. 附加区：GitHub 热力图、技术统计、运行状态

## 本轮结构决定

本轮优先采用三段式结构：

1. Hero：身份、当前状态、去往各内容区的主要入口
2. Workbench：最近动态、重点项目、内容导航卡片
3. Signals：关注主题、轻量统计、与首页/关于页的关系说明

其中：

- `/me` 视觉上应比首页更像“工作台”
- `/` 保留为站点欢迎页与总览入口
- `/about` 更适合承载方法论、边界与项目背景

## 当前实现结果

- `/me` 已按工作台方向落地，首页与其视觉职责已分开
- 内容入口、最近动态、重点项目、关注主题和轻量统计已形成三段式结构
- 统计模块保持克制，仍然服务于内容导航而不是监控展示

## 后续优化方向

- 强化主页与其他内容页的视觉区分
- 让统计模块更克制，不抢正文入口
- 让页面既能浏览，也能快速跳转到内容
