# 第三版产品需求

## 1. 第三版定位

第三版不再只是补页面链路，而是把这个站点从“可浏览的个人内容档案”推进到“更适合长期写作、整理、检索和公开传播的内容系统”。

第二版已经完成了：

- `notes / projects` 详情链路
- 标签归档与站内搜索
- 页面结构、内容入口与基础 SEO 提升

第三版要解决的问题是：

- 内容之间仍然偏孤立
- 搜索已可用，但结果质量和筛选能力还不够强
- 内容模型还偏轻，长期维护信息密度不足
- 发布质量与内容治理仍偏手工

## 2. 第三版总体策略

第三版建议继续坚持：

- 以现有 `blog / notes / projects` 三类内容为核心
- 不引入数据库和后台 CMS
- 优先做长期维护价值高、复杂度可控的能力

第三版不建议一开始就转向评论、用户、权限、富文本后台这类平台化系统。

## 3. 第三版目标

第三版需要完成以下目标：

1. 提升内容组织能力
2. 提升内容关联与发现能力
3. 提升内容发布与维护效率
4. 提升公开传播与长期迭代质量

## 4. 第三版建议范围

### 4.1 必做范围

1. 内容关联系统
2. 归档与索引升级
3. 搜索结果质量升级
4. 内容模型增强
5. 发布前内容校验与治理

### 4.2 建议增强

1. 标签总览页
2. 年度归档页
3. 系列页或主题页
4. 更完整的站点公开页面，如 `now / changelog / uses`

## 5. 页面与功能范围

第三版正式实现重点建议为：

1. `/tags`
2. `/archive`
3. `/series/:slug` 或主题页
4. `/search` 进阶版
5. 内容详情页上的相关文章/相关笔记/相关项目模块

并对以下已有页面继续升级：

1. `/blog/:slug`
2. `/notes/:slug`
3. `/projects/:slug`
4. `/`

## 6. 内容模型方向

第三版不改变三类内容模型的基本边界，但会增强字段密度与组织能力。

### `blog`

- 增加系列、阅读时长、相关文章
- 增强标签与主题表达

### `notes`

- 增加阶段、来源、实验状态等字段
- 更清楚地区分学习记录与结论型笔记

### `projects`

- 增加背景、角色、时间线、结果、复盘信息
- 更适合展示长期项目过程

## 7. 第三版不做

第三版仍明确不做以下内容：

1. 评论系统
2. 登录注册
3. 用户系统
4. 数据库
5. 后台管理系统
6. 富文本编辑器
7. 多角色权限

## 8. 成功标准

第三版完成后，应满足以下标准：

1. 站内内容不再是孤立页面，而能形成关联网络
2. 搜索不只是可用，而是更容易找到真正相关内容
3. 新增与维护内容的成本更低
4. 内容公开传播与长期演进质量明显高于第二版
