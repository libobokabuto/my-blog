# 第三版产品需求

## 1. 第三版定位

第三版不再只是补页面链路，而是把这个站点从“可浏览的个人内容档案”推进到“更适合长期写作、整理、检索和公开传播的内容系统”。

第二版已经完成了：

- `notes / projects` 详情链路
- 标签归档与站内搜索
- 页面结构、内容入口与基础 SEO 提升

第三版要解决的问题是：

- 内容之间仍然偏孤立
- 搜索已可用，但结果质量和筛选能力还不够强
- 内容模型还偏轻，长期维护信息密度不足
- 发布质量与内容治理仍偏手工

## 2. 第三版总体策略

第三版建议继续坚持：

- 以现有 `blog / notes / projects` 三类内容为核心
- 不引入数据库和后台 CMS
- 优先做长期维护价值高、复杂度可控的能力

第三版不建议一开始就转向评论、用户、权限、富文本后台这类平台化系统。

## 3. 第三版目标

第三版需要完成以下目标：

1. 提升内容组织能力
2. 提升内容关联与发现能力
3. 提升内容发布与维护效率
4. 提升公开传播与长期迭代质量

## 4. 第三版建议范围

### 4.1 必做范围

1. 个人主页
2. 内容关联系统
3. 归档与索引升级
4. 搜索结果质量升级
5. 内容模型增强
6. 发布前内容校验与治理

### 4.2 建议增强

1. 标签总览页
2. 年度归档页
3. 系列页或主题页
4. 更完整的站点公开页面，如 `now / changelog / uses`
5. GitHub 贡献热力图、技术统计、运行状态等附加模块

## 5. 页面与功能范围

第三版正式实现重点建议为：

1. `/me` 或新的个人主页入口
2. `/tags`
3. `/archive`
4. `/series/:slug` 或主题页
5. `/search` 进阶版
6. 内容详情页上的相关文章/相关笔记/相关项目模块

并对以下已有页面继续升级：

1. `/blog/:slug`
2. `/notes/:slug`
3. `/projects/:slug`
4. `/`
5. `/about`

## 5.1 个人主页定位

第三版的个人主页不应只是一个“服务器监控面板”或“GitHub 数据看板”，而应是：

- 个人内容站中的公开工作台
- 展示“我是谁、我在做什么、我最近写了什么、我最近学了什么”
- 连接内容、项目、学习、版本变化和少量统计的枢纽页

## 5.2 个人主页建议内容

个人主页建议优先包含：

1. 当前状态 / Now
2. 最近发布与最近更新
3. 当前重点项目快照
4. 当前关注主题或标签入口
5. 轻量统计，如博客数、笔记数、项目数、最近更新次数

## 5.3 个人主页建议增强

以下能力适合作为个人主页的增强模块，而不是主体：

1. GitHub 提交热力图
2. 技术语言或仓库分布统计
3. 站点运行状态或 uptime
4. 服务器资源监测

第三版不建议一开始就把主页重心放在实时监控图表上，否则容易偏离“个人内容站”主方向。

## 6. 内容模型方向

第三版不改变三类内容模型的基本边界，但会增强字段密度与组织能力。

### `blog`

- 增加系列、阅读时长、相关文章
- 增强标签与主题表达

### `notes`

- 增加阶段、来源、实验状态等字段
- 更清楚地区分学习记录与结论型笔记

### `projects`

- 增加背景、角色、时间线、结果、复盘信息
- 更适合展示长期项目过程

## 7. 第三版不做

第三版仍明确不做以下内容：

1. 评论系统
2. 登录注册
3. 用户系统
4. 数据库
5. 后台管理系统
6. 富文本编辑器
7. 多角色权限

## 8. 成功标准

第三版完成后，应满足以下标准：

1. 站内内容不再是孤立页面，而能形成关联网络
2. 搜索不只是可用，而是更容易找到真正相关内容
3. 个人主页能作为站点的公开导航枢纽
4. 新增与维护内容的成本更低
5. 内容公开传播与长期演进质量明显高于第二版
