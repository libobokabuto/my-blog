# my-blog v1.0 部署说明
## 1. 目标
这份文档用于把第五版收尾后的 `my-blog` 部署到服务器，并说明服务器需要准备什么、为什么需要、推荐怎么配置。

## 2. v1.0 已包含能力
1. SSR 站点首页与公开内容页
2. `rss.xml` 与 `sitemap.xml`
3. `/admin`
4. `/admin/content`
5. `/admin/search`
6. `/admin/stats`
7. `/admin/tasks`
8. `/admin/sync`
9. MySQL 持久化搜索、统计、任务、同步记录
10. Redis 搜索运行态记录

## 3. v1.0 明确不包含
1. 完整用户系统
2. 多角色权限
3. 评论系统
4. 开放平台
5. 复杂工作流编排
6. 独立 worker 与队列系统

## 4. 服务器上需要准备什么
### 4.1 必需项
1. 一台可运行 Rust 服务的服务器
   原因：项目是 Leptos SSR + Axum，服务端必须常驻运行
2. MySQL 8.x
   原因：搜索文档、重建记录、统计快照、任务记录、同步记录都落在 MySQL
3. Redis
   原因：搜索运行态状态写入 Redis
4. 一个站点域名或至少一个可访问地址
   原因：`SITE_URL` 影响页面 canonical、RSS、sitemap
5. 一个反向代理
   原因：推荐由 Nginx 对外暴露 80/443，再转发到应用端口

### 4.2 推荐项
1. Linux 服务器
   原因：部署与守护进程管理更直接
2. HTTPS
   原因：正式站点、RSS、搜索引擎收录都更稳定
3. 非 root 的 MySQL 业务账号
   原因：比直接使用 root 更安全

## 5. 推荐目录结构
```text
/srv/my-blog
├─ app/
├─ client/
├─ content/
├─ deploy/
├─ server/
├─ static/
├─ style/
├─ Cargo.toml
├─ Cargo.lock
└─ target/
```

## 6. 内容以后分别放到哪里
1. 博客内容放在 `content/blog/*.md`
2. 笔记内容放在 `content/notes/*.md`
3. 项目内容放在 `content/projects/*.md`

这三个目录继续作为 `v1.0` 的正式内容入口，不切到数据库写内容。

## 7. 必要环境变量
1. `SITE_URL`
   作用：站点正式地址，用于 canonical、RSS、sitemap
   示例：`https://blog.example.com`
2. `BLOG_DATABASE_URL`
   作用：连接 MySQL
   示例：`mysql://blog_user:password@127.0.0.1:3306/my-blog`
3. `BLOG_REDIS_URL`
   作用：连接 Redis
   示例：`redis://127.0.0.1:6379/`
4. `RUST_LOG`
   作用：控制运行日志
   示例：`server=info,tower_http=info`

## 8. 本地运行命令
### 8.1 先准备环境变量
PowerShell 示例：
```powershell
$env:SITE_URL='http://127.0.0.1:3000'
$env:BLOG_DATABASE_URL='mysql://root:your-password@127.0.0.1:3306/my-blog'
$env:BLOG_REDIS_URL='redis://127.0.0.1:6379/'
$env:RUST_LOG='server=info,tower_http=info'
```

### 8.2 开发检查
```powershell
cargo check
```

### 8.3 直接启动服务
```powershell
cargo run -p server --bin server
```

## 9. 生产构建与启动
### 9.1 构建
```powershell
cargo build -p server --bin server --release
```

### 9.2 启动
```powershell
./target/release/server.exe
```

如果部署到 Linux，请按实际目标平台构建对应二进制，不要直接使用当前 Windows 产物。

## 10. 反向代理
仓库中已提供示例文件：
- `deploy/nginx.my-blog.conf.example`
- `deploy/my-blog.service.example`

推荐做法：
1. Nginx 对外监听 80/443
2. 反向代理到 `127.0.0.1:3000`
3. 正式环境使用 HTTPS

## 11. 数据库建议
### 11.1 数据库
推荐数据库名：`my-blog`

### 11.2 业务账号
推荐创建单独账号，例如：
- 用户名：`blog_user`
- 权限：仅授予 `my-blog` 库所需权限

### 11.3 当前使用到的表
1. `search_documents`
2. `search_rebuild_runs`
3. `stats_snapshots`
4. `task_runs`
5. `sync_sources`
6. `sync_runs`

这些表当前由应用启动后按需自举创建。

## 12. 上线前最少检查项
1. 首页可访问
2. `/rss.xml` 可访问
3. `/sitemap.xml` 可访问
4. `/admin/search` 显示 MySQL / Redis 已连接
5. 搜索重建至少执行一次
6. `/admin/stats` 有快照
7. `/admin/tasks` 有任务记录
8. `/admin/sync` 有同步源和同步记录

## 13. 当前 v1.0 收尾结论
第五版的目标不是扩平台，而是让第四版已经进入结构的后端能力真正可运行、可验证、可部署。
